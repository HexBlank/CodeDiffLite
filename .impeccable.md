## Design Context

### Users
Developers, code reviewers, and technical administrators who need to review source code modifications efficiently. They operate in a high-focus environment where precision, clarity, and speed are critical.

### Brand Personality
Precision meets Elegance. The interface should feel high-end, premium, and atmospheric (优雅大气) while remaining structurally sound for technical work.

### Aesthetic Direction
**Modern Premium (Elegant & Smooth)**. 
- **Corners & Geometry**: Embrace modern, smooth aesthetics with generous rounded corners (e.g., `0.75rem` - `1rem`) to ensure the interface is comfortable, inviting, and visually grand.
- **Lighting & Depth**: Rather than sharp, brutalist lines or cheap generic blurs, use refined, multi-layered soft drop shadows and subtle border translucency to define elements gracefully. 
- **Colors**: OKLCH palettes with a delicate, clean slate/blue tint. The contrast is balanced—neither muddy nor aggressively sharp.
- **Typography**: Clean, balanced sans-serifs that communicate sophistication without being generic.

### Design Principles
1. **Elegant Smoothness**: Use rounded corners where appropriate. Do not force sharp angles; focus on comfortable spatial flow.
2. **Premium Depth over Blur**: Define elevation through high-quality, physics-based shadow interpolations rather than simplistic `backdrop-blur`.
3. **Intentional Tinting**: All neutrals share a subconscious OKLCH hue vector (clean slate), providing a feeling of bespoke tailoring.
4. **Fluid Motion**: Transitions should be buttery smooth and elegant, using refined `ease-out` curves that feel natural, not jarringly fast.
